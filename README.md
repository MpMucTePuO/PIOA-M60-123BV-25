# База данных сотрудников

## Основные модули

`src/db/backend/database.py` содержит общий интерфейс базы данных:

- `Database` задаёт методы работы с сотрудниками.
- `Memory` и `FileDatabase` наследуются от `Database`.

`src/db/backend/record.py` содержит класс записи:

- `Record` описывает одну запись сотрудника.

`src/db/backend/table.py` содержит класс таблицы:

- `Table` хранит сотрудников и проверяет операции с записями.

`src/db/backend/memory.py` содержит базу данных в оперативной памяти:

- `Memory` реализует интерфейс `Database`.
- Данные хранятся только до завершения программы.

`src/db/backend/file.py` содержит файловую базу данных:

- `FileDatabase` реализует интерфейс `Database`.
- Данные хранятся в файле `employees.json`.
- В файле сохраняется структура таблицы и список записей.

`src/db/backend/errors.py` содержит пользовательские исключения:

- `DatabaseError`
- `EmployeeTableError`
- `InvalidEmployeeDataError`
- `DuplicateEmployeeError`
- `EmployeeNotFoundError`
- `FileDatabaseError`
- `InvalidStorageDataError`

`src/db/tui.py` содержит текстовый интерфейс, который отвечает за меню,
ввод данных и вывод результатов.

При запуске программы пользователь выбирает базу данных:

- база в оперативной памяти работает только до завершения программы;
- файловая база сохраняет данные в папке `data`;
- если файла `employees.json` нет, он создаётся автоматически.

## Функциональность

Программа поддерживает:

- добавление сотрудников;
- поиск сотрудников;
- обновление данных сотрудника;
- удаление сотрудников;
- работу с базой в оперативной памяти;
- работу с файловой базой данных;
- фильтрацию сотрудников по `id`, имени, должности, отделу и зарплате;
- проверку уникальности `id`;
- проверку обязательных полей;
- проверку корректности зарплаты;
- сохранение данных файловой базы после завершения программы;
- обработку повреждённого JSON-файла и неправильной структуры данных.

## Формат файловой базы

Файловая база создаёт папку `data` и JSON-файл:

- `employees.json`

В файле хранится структура таблицы и список записей:

```json
{
  "columns": [
    "employee_id",
    "name",
    "position",
    "department",
    "salary"
  ],
  "records": []
}
```

## Запуск проекта

```bash
python -m src.db
```

После запуска необходимо выбрать тип базы данных:

```text
1. Оперативная память
2. Файловая база данных
```
