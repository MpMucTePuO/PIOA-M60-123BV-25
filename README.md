Программа хранит сведения о сотрудниках компании в оперативной памяти.
Можно добавлять, искать, изменять и удалять записи. Реализация построена
на классах, а для работы с программой используется текстовое меню.

## Структура проекта

```text
src/
└── db/
    ├── __main__.py
    ├── tui.py
    └── backend/
        ├── errors.py
        ├── record.py
        ├── table.py
        └── memory.py
tests/
└── test_memory.py
```

### Классы

- `Record` хранит данные одного сотрудника.
- `Table` хранит список записей, проверяет данные и выполняет добавление,
  поиск, изменение и удаление сотрудников.
- `Memory` создаёт таблицу сотрудников в оперативной памяти и содержит
  методы для работы с ней.
- `errors.py` содержит пользовательские исключения для ошибок таблицы.

В классе `Memory` находятся методы `create_employee`, `select_employees`,
`update_employee` и `delete_employee`.

### Данные сотрудника

Каждая запись содержит:

- ID сотрудника;
- имя;
- должность;
- отдел;
- зарплату.

ID и зарплата не могут быть отрицательными. Имя и должность не могут быть
пустыми. Сотрудники с одинаковыми ID не добавляются.

Ошибки добавления, изменения и удаления оформлены через пользовательские
исключения: `InvalidEmployeeDataError`, `DuplicateEmployeeError` и
`EmployeeNotFoundError`.

## Запуск программы

Из корня проекта:

```bash
python -m src.db
```

## Запуск тестов

```bash
python -m unittest discover -s tests -v
```

Проверка покрытия без модуля текстового интерфейса:

```bash
python -m pip install coverage
python -m coverage run --source=src.db.backend -m unittest discover -s tests
python -m coverage report
```
